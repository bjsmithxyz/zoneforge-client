# ZoneForge Client — Claude Code Context

> See the root `CLAUDE.md` (one level up, in the umbrella repo) for project summary, repo layout, stack, and daily dev loop.

## This Submodule

Unity 2022.3 LTS (2022.3.62f3) project using URP. C# scripting throughout. Communicates with the SpacetimeDB server module via the SpacetimeDB C# SDK.

## Project Layout

```
Assets/
├── Scripts/
│   ├── Data/        ← ScriptableObject definitions (WorldData, ZoneVisualData)
│   ├── Network/     ← SpacetimeDBManager singleton — connect/subscribe/tick
│   ├── Zone/        ← ZoneController and runtime zone logic
│   ├── Editor/      ← Unity Editor-only scripts (excluded from builds)
│   └── autogen/     ← Generated by `spacetime generate` — NEVER edit manually
└── Art/
    ├── Models/
    ├── Tiles/
    └── Materials/
        ├── Graphs/      ← Material Maker .mmg source files
        ├── Exports/     ← Albedo, normal, roughness maps
        └── Parameters/  ← Material Maker presets
```

## Key Conventions

**Editor scripts** — must live in `Assets/Scripts/Editor/` to be excluded from game builds. Use `[MenuItem("ZoneForge/...")]` for custom menu entries.

**Autogen** — `Assets/Scripts/autogen/` is fully generated. Never edit files here. Regenerate after any server schema change:

```bash
# Run from this directory (client/)
spacetime generate --lang csharp \
  --out-dir Assets/Scripts/autogen \
  --bin-path ../server/spacetimedb/target/wasm32-unknown-unknown/release/zoneforge_server.wasm
```

**VS Code settings** — Unity auto-generates `.vscode/settings.json` when you run Assets → Open C# Project. Do not overwrite it.

**Database name** — `zoneforge-server` (lowercase with dashes).

## SpacetimeDB C# SDK Patterns

### Connection (in SpacetimeDBManager)

```csharp
_conn = DbConnection.Builder()
    .WithUri("http://localhost:3000")
    .WithModuleName("zoneforge-server")
    .OnConnect(OnConnect)
    .OnConnectError(OnConnectError)
    .OnDisconnect(OnDisconnect)
    .Build();
```

### Subscribe in OnConnect — not before

```csharp
void OnConnect(DbConnection conn, Identity identity, string token)
{
    conn.SubscriptionBuilder()
        .OnApplied(OnSubscriptionApplied)
        .Subscribe(new[] { "SELECT * FROM player", "SELECT * FROM zone" });
}
```

### Register callbacks in OnSubscriptionApplied — not before

```csharp
void OnSubscriptionApplied(SubscriptionEventContext ctx)
{
    ctx.Db.Player.OnInsert += OnPlayerInserted;
    ctx.Db.Player.OnUpdate += OnPlayerUpdated;
}
```

### FrameTick — must be called every Update

```csharp
void Update()
{
    _conn?.FrameTick();
}
```

Without `FrameTick()`, no SDK callbacks will fire.

### Calling reducers

```csharp
// Generated reducer classes live in autogen/
Reducer.CreatePlayer(ctx, "PlayerName");
Reducer.MovePlayer(conn, newX, newY);
```

## Common Mistakes

| Mistake | Fix |
| ------- | --- |
| Callbacks not firing | Ensure `_conn.FrameTick()` is called in `Update()` |
| Autogen types unresolved after schema change | Run `spacetime generate`, then Assets → Reimport All |
| Subscribing in `Start()` before `OnConnect` fires | Always subscribe inside the `OnConnect` callback |
| Editing files in `autogen/` | Never — they are overwritten on next `spacetime generate` |
| Missing `public` on a server table | Clients can't subscribe — fix in server Rust code |

## Unity-Specific Notes

- **Play mode** (`Ctrl+P`) — connects to local SpacetimeDB, full game loop
- **Console** (`Window → General → Console`) — connection and callback logs appear here
- **Reimport All** (`Assets → Reimport All`) — use if autogen imports don't trigger automatically
- **Unity generates `.csproj`/`.sln`** — run Assets → Open C# Project once after setup, and again after adding Assembly Definition files
